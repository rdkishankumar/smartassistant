# Notes on Implementing Message Roles in Spring AI

## Project Setup
- Project: "SpringAI" (similar to hello world app).
- **ChatController**: Inject ChatClient via ChatClient.builder().build().
- Method: Return `chatClient.prompt(message).call().content()`.
- **application.properties**: OpenAI API key configured.

## Behind the Scenes: Prompt and Messages
- `prompt(String)`: Creates a `UserMessage` (default user role).
- **Message Hierarchy**:
    - `Message` interface: Represents messages in chat.
    - `AbstractMessage`: Base class.
    - Implementations: `UserMessage`, `SystemMessage`, `AssistantMessage`, `ToolResponseMessage` (Function/Tool role).
- `Prompt` class: Accepts `List<Message>` or individual messages via constructors.
- Fluent API: `chatClient.prompt().system("instructions").user("message").call().content()`.

## Adding System and User Messages
- Use fluent API to specify roles:
    - `system(String)`: Instructions for LLM behavior (e.g., restrict to HR assistant).
- Example System Message: "You are an internal HR assistant. Help with HR policies (leave, hours, benefits, conduct). Decline other topics."
- User Message: End user's question (e.g., "What is your name and which model you are using?").

## Testing
- Postman request to `/api/chat?message=What is your name...`:
    - Response: Restricted (e.g., "I'm here to assist with HR policy questions...").
- Request: "How many leaves can I take?":
    - Response: HR-related (generic; will improve with company data training later).

## Key Insights
- System messages control LLM behavior and restrict responses.
- Spring AI handles role differentiation automatically.
