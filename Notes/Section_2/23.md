# Detailed Notes on Configuring ChatOptions in Spring AI (Demo)

## 1. Configuring Default ChatOptions on ChatClient Builder

### Code Example
```java name=ChatClientConfig.java
@Bean
public ChatClient chatClient(ChatClient.Builder builder) {
    ChatOptions chatOptions = ChatOptions.builder()
        .model("gpt-4.1-mini")
        .maxTokens(100)
        .temperature(0.8)
        .build();

    return builder
        .defaultOptions(chatOptions)
        .defaultAdvisors(List.of(
            new SimpleLoggerAdvisor(),
            new TokenUsageAuditAdvisor()
        ))
        .build();
}
```

- `defaultOptions()`: Applies ChatOptions to **all REST APIs** using this ChatClient.
- `ChatOptions.builder()`: Generic builder; works across all LLM providers.

## 2. Testing Default ChatOptions

### Model Verification
- Check console logs for the model name used.
- Confirmed: `gpt-4.1-mini` is used (as configured).

### maxTokens Behavior
| maxTokens Value | Result |
|----------------|--------|
| `100` | Full response generated (60 tokens used — within budget) |
| `10` | Response **abruptly terminated** at 10 tokens |

- ⚠️ Don't set maxTokens too low — responses will be cut off mid-sentence.
- Remove or set to reasonable value for general use in course demos.

## 3. Per-Request Options with Provider-Specific Class

### Using `OpenAiChatOptions` for a Specific REST API
```java name=PromptStuffingController.java
return chatClient.prompt()
    .system(systemPromptTemplate)
    .user(message)
    .options(OpenAiChatOptions.builder()
        .model(OpenAiApi.ChatModel.GPT_4_O_LATEST)  // Using enum constant
        .build())
    .call()
    .content();
```

- `options()`: Applies only to **this specific request** (overrides default options).
- `OpenAiChatOptions`: Provides OpenAI-specific parameters beyond generic ChatOptions.

## 4. Generic vs. OpenAI-Specific Options Comparison

### Generic `ChatOptions`
```java name=GenericOptions.java
ChatOptions.builder()
    .model("gpt-4.1-mini")
    .temperature(0.8)
    .maxTokens(100)
    .frequencyPenalty(0.5)
    .build();
```

### OpenAI-Specific `OpenAiChatOptions`
```java name=OpenAiOptions.java
OpenAiChatOptions.builder()
    .model(OpenAiApi.ChatModel.GPT_4_O_LATEST)  // Enum constant or String
    .frequencyPenalty(0.5)
    // OpenAI-exclusive options:
    .webSearchOptions(...)
    .logitBias(...)
    .logProbs(true)
    .maxCompletionTokens(500)
    .metadata(...)
    .outputAudio(...)
    .parallelToolCalls(true)
    .reasoningEffort(...)
    .httpHeaders(...)
    .build();
```

### Two Ways to Specify Model
```java name=ModelSelection.java
// Option 1: String format
.model("gpt-4o-latest")

// Option 2: Enum constant (recommended; type-safe)
.model(OpenAiApi.ChatModel.GPT_4_O_LATEST)
```

## 5. Configuring ChatOptions via application.properties

### Property-Based Configuration
```properties name=application.properties
# Generic chat options
spring.ai.chat.options.model=gpt-4.1-mini
spring.ai.chat.options.temperature=0.8
spring.ai.chat.options.max-tokens=500
spring.ai.chat.options.frequency-penalty=0.5

# OpenAI-specific options
spring.ai.openai.chat.options.model=gpt-4.1-mini
spring.ai.openai.chat.options.temperature=0.8
```

### When to Use Properties vs. Code

| Configuration Method | When to Use |
|---------------------|-------------|
| **Java code** (`ChatOptions.builder()`) | Static values; same across environments |
| **application.properties** | Values change per environment (dev/staging/prod) |
| **Spring Boot Profiles** | Different model/options per deployment environment |

### Example: Multi-Environment Setup
```properties name=application-dev.properties
spring.ai.openai.chat.options.model=gpt-4.1-mini
spring.ai.openai.chat.options.temperature=0.9
```

```properties name=application-prod.properties
spring.ai.openai.chat.options.model=gpt-4o
spring.ai.openai.chat.options.temperature=0.7
```

## 6. Method Scope Summary

| Method | Scope | Class |
|--------|-------|-------|
| `defaultOptions()` on builder | All requests via ChatClient | `ChatClient.Builder` |
| `options()` on prompt | Single specific request | Prompt chain |

---

## 7. Best Practices

| Practice | Explanation |
|---------|-------------|
| Don't configure all options | Default values are reasonable; only override when needed |
| Remove `maxTokens` for demos | Prevents abrupt response truncation during exploration |
| Use enum constants for models | Type-safe; avoids typos (e.g., `OpenAiApi.ChatModel.GPT_4_O`) |
| Use properties for env-specific values | Enables Spring Boot Profiles for multi-environment deployments |
| Read provider class documentation | Each option is documented; helps understand purpose |

---

## 8. Summary

| Concept | Key Point |
|---------|-----------|
| `ChatOptions` | Generic tuning panel for all LLM providers |
| `OpenAiChatOptions` | OpenAI-specific options (extra parameters) |
| `defaultOptions()` | Applied to all requests via ChatClient bean |
| `options()` | Applied to specific request only |
| `maxTokens` | Hard budget on response length; use carefully |
| Property-based config | Recommended for environment-specific settings |
| Model selection | Use enum constants for type safety |
