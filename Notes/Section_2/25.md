# Detailed Notes on Streaming Responses in Spring AI

## 1. What Is Streaming?

### Normal Call (`call()`)
- Client sends request → **waits** for complete response → displays all at once.
- Thread is **blocked** until the full response arrives.
- Similar to downloading a file before playing it.

### Streaming (`stream()`)
- Client sends request → receives response **chunks as they are generated**.
- Thread is **not blocked**; processes each chunk and releases the thread until the next chunk arrives.
- Similar to video streaming — plays as it loads.
- Think of Flux as a **conveyor belt** delivering products asynchronously.

---

## 2. Creating the StreamController

```java name=StreamController.java
@RestController
@RequestMapping("/api")
public class StreamController {

    private final ChatClient chatClient;

    public StreamController(ChatClient.Builder chatClientBuilder) {
        this.chatClient = chatClientBuilder.build();
    }

    @GetMapping("/stream")
    public Flux<String> stream(@RequestParam String message) {
        return chatClient.prompt()
                .user(message)
                .stream()       // ← use stream() instead of call()
                .content();     // returns Flux<String>
    }
}
```

---

## 3. `call()` vs `stream()` Method Comparison

| Aspect | `call()` | `stream()` |
|--------|---------|-----------|
| Response delivery | All at once after complete | Chunk by chunk as generated |
| Thread behavior | Blocked until response complete | Non-blocking; thread freed between chunks |
| Return type | `String`, `ChatResponse`, `ChatClientResponse` | `Flux<String>`, `Flux<ChatResponse>`, `Flux<ChatClientResponse>` |
| Use case | Simple request-response | Real-time response display (e.g., ChatGPT-like UI) |
| Testing tool | Postman (supported) | Browser or SSE-compatible client |

---

## 4. Flux in Spring Reactive

### What Is Flux?
- Part of **Spring Reactive / Project Reactor** ecosystem.
- Represents a **stream of 0 to N elements** emitted asynchronously.
- Non-blocking: processes each emitted item and frees the thread.
- Completes when the LLM sends a **completion signal** (end of response).

### How It Works with LLM Streaming
```
LLM generates token 1 → Flux emits → Client receives → Thread freed
LLM generates token 2 → Flux emits → Client receives → Thread freed
...
LLM finishes          → Flux completes → Stream ends
```

### Available Stream Methods
```java name=StreamMethods.java
// Returns Flux<String> - just the text content
chatClient.prompt().user(message).stream().content();

// Returns Flux<ChatResponse> - full response object per chunk
chatClient.prompt().user(message).stream().chatResponse();

// Returns Flux<ChatClientResponse> - full client response per chunk
chatClient.prompt().user(message).stream().chatClientResponse();
```

## 5. Testing Streaming

### Why Not Postman?
- Postman **does not support** streaming responses natively.
- Use a **browser** or SSE-compatible client instead.

### Browser Test
- Stream API: `http://localhost:8080/api/stream?message=Tell me about all the HR policy details`
    - Response appears **progressively** as tokens arrive.
- Chat API: `http://localhost:8080/api/chat?message=Tell me about all the HR policy details`
    - Browser **waits** → displays full response at once.

---

## 6. When to Use Streaming

| Scenario | Recommendation |
|----------|---------------|
| ChatGPT-like UI (real-time text display) | ✅ Use `stream()` |
| Long responses (avoid timeout) | ✅ Use `stream()` |
| Simple request-response | ✅ Use `call()` |
| Postman testing | ✅ Use `call()` |
| Background processing | ✅ Use `call()` |

---

## 7. Key Rules for Streaming

- Replace `call()` with `stream()` in the prompt chain.
- Wrap return type with `Flux<>` (e.g., `Flux<String>`).
- Import `reactor.core.publisher.Flux`.
- Use browser or SSE client for testing (not Postman).
- `SimpleLoggerAdvisor` aggregates chunks before logging (as discussed in advisor notes).

---

## 8. Summary

| Concept | Key Point |
|---------|-----------|
| `stream()` | Non-blocking; returns response as Flux chunks |
| `call()` | Blocking; returns complete response at once |
| `Flux` | Spring Reactive type for async stream of elements |
| Testing | Use browser for stream; Postman for call |
| Thread behavior | stream() frees thread between chunks |
| Return types | All return types wrapped in Flux for stream() |
