# Detailed Notes on Prompt Templates in Spring AI

## 1. Why Prompt Templates?

### The Problem with Simple Strings
- Real enterprise applications often send prompts of 200-300+ lines.
- Complex scenarios require more details to be passed to the LLM.
- Hardcoding prompts as plain strings inside business logic classes is:
    - Messy and hard to maintain.
    - Mixes business logic with prompt text (bad separation of concerns).
    - Not reusable.

### Benefits of Prompt Templates
- **Simplifies** complex prompt construction.
- **Reusable** across the application.
- **Maintainable** – prompts are separated from business logic.
- **Parameterized** – supports dynamic placeholders.
- **Clean separation** of business logic and prompt text.

---

## 2. What Is a Prompt Template?

- A **String** containing:
    - **Static content** (fixed instructions).
    - **Dynamic placeholders** (e.g., `{customerName}`, `{customerMessage}`).
- Placeholders are filled at runtime with actual values.

---

## 3. Defining Prompt Templates

### Approach 1: Inline (Basic – Not Recommended)
- Define as a String constant directly in the controller/business class.
- **Pros**: Simple for short prompts.
- **Cons**: Makes business logic clumsy; hard to maintain for large prompts.

```java name=ChatController.java
private static final String promptTemplate = """
    A customer named {customerName} sent the following message:
    {customerMessage}
    Write a polite and helpful email response addressing the issue.
    Maintain a professional tone and provide reassurance.
    Respond as if you are writing the email body only.
    Don't include subject and signature.
    """;
```

### Approach 2: External File (Recommended)
- Create a folder under `resources`: `promptTemplates/`.
- Create files with `.st` extension (StringTemplate).
- Define prompts in the `.st` file with static content and placeholders.
- Load via `@Value` annotation into a `Resource` variable.

```plaintext name=src/main/resources/promptTemplates/emailResponse.st
A customer named {customerName} sent the following message:
{customerMessage}
Write a polite and helpful email response addressing the issue.
Maintain a professional tone and provide reassurance.
Respond as if you are writing the email body only.
Don't include subject and signature.
```

---

## 4. Using Prompt Templates in Code

### Loading the Template
```java name=PromptTemplateController.java
@Value("classpath:promptTemplates/emailResponse.st")
private Resource userPromptTemplate;
```

### Using the Template with ChatClient Fluent API
```java name=PromptTemplateController.java
chatClient.prompt()
    .system("You are a professional customer service assistant which helps drafting email responses to improve the productivity of the customer support team.")
    .user(promptTemplateSpec -> promptTemplateSpec
        .text(userPromptTemplate)
        .param("customerName", customerName)
        .param("customerMessage", customerMessage)
    )
    .call()
    .content();
```

- `text()`: Accepts the Resource (or String) containing the template.
- `param()`: Sets placeholder key-value pairs.
- Multiple `param()` calls for multiple placeholders.

## 5. Demo Scenario

### REST API: Email Response Generator
- **Endpoint**: `GET /api/email`
- **Parameters**: `customerName`, `customerMessage`
- **System Message**: "You are a professional customer service assistant..."
- **User Template**: Fills `{customerName}` and `{customerMessage}` dynamically.

### Example Request
```
GET /api/email?customerName=Madan&customerMessage=I'm not able to access my account, it says account locked. Please help.
```

### Example Response
```
Dear Madan,
Thank you for reaching out to us regarding the issue with your account access...
[Professional email body generated by LLM]
```

### Business Value
- Customer support executives don't need to type full responses.
- They can review, edit if needed, add name/signature, and send.
- Dramatically improves productivity of customer support teams.

## 6. Methods Supporting Prompt Templates

All these ChatClient fluent API methods support prompt templates (Resource or String):
- `user()` – for user role messages.
- `system()` – for system role messages.
- `defaultSystem()` – default system message for all requests.
- `defaultUser()` – default user message for all requests.

---

## 7. Custom Placeholder Delimiters

- Default placeholders use curly braces: `{placeholder}`.
- Can use custom delimiters (e.g., `<placeholder>`).
- Implement the `TemplateRender` interface and override `startDelimiterToken()` and `endDelimiterToken()`.

```java name=CustomTemplateRenderer.java
// Example: Using < > as delimiters
public class CustomTemplateRenderer implements TemplateRenderer {
    @Override
    public String startDelimiterToken() { return "<"; }

    @Override
    public String endDelimiterToken() { return ">"; }
}
```

- Reference: [Spring AI Official Documentation – Prompts](https://docs.spring.io/spring-ai/reference/api/prompt.html)

## 8. Summary

| Approach | When to Use | Pros | Cons |
|----------|-------------|------|------|
| Inline String constant | Short/simple prompts (2-3 lines) | Quick to implement | Clumsy for large prompts |
| External `.st` file | Complex/long prompts (recommended) | Clean separation, maintainable, reusable | Slightly more setup |

---

## 9. Key Annotations and Classes
| Element | Purpose |
|---------|---------|
| `@Value("classpath:...")` | Load prompt template file into Resource |
| `Resource` (springframework.core.io) | Holds template file content |
| `.text()` | Pass template content to prompt builder |
| `.param(key, value)` | Fill placeholders in template |
| `TemplateRender` interface | Customize placeholder delimiters |
