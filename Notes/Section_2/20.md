# Detailed Notes on Configuring Advisors in Spring AI

## 1. Configuring Default Advisors on ChatClient

### Three Overloaded `defaultAdvisors()` Methods
- Single advisor object.
- List of advisor objects.
- Lambda expression (Consumer Functional Interface) for dynamic configuration.

### Example: Configuring SimpleLoggerAdvisor
```java name=ChatClientConfig.java
@Bean
public ChatClient chatClient(ChatClient.Builder builder) {
    return builder
        .defaultAdvisors(new SimpleLoggerAdvisor())  // applies to all requests
        .build();
}
```

---

## 2. SimpleLoggerAdvisor Deep Dive

### Purpose
- Logs all **request** and **response** messages for every LLM interaction.

### Interfaces Implemented
- `CallAdvisor` – for synchronous (blocking) calls.
- `StreamAdvisor` – for streaming responses.
- Implementing both means the advisor supports both normal and streaming calls.

### Key Methods

#### `adviceCall()`
```
1. Log request (logRequest())
2. Forward to next advisor in chain (or LLM if no more advisors)
3. Receive LLM response
4. Log response (logResponse())
5. Return response
```

#### `adviceStream()`
```
1. Log request (logRequest())
2. Forward to LLM (streaming)
3. Aggregate all streamed responses
4. Log aggregated response at the end
5. Return Flux<ChatClientResponse>
```

### Why Aggregate in Stream Mode?
- Streaming sends response chunks continuously.
- Logging each chunk separately would be excessive.
- Aggregation logs the complete response once at the end.

### Logging Level
- Uses `logger.debug()` (debug level).
- Debug logging is **disabled by default**.
- Must enable explicitly in `application.properties`.

```properties name=application.properties
logging.level.org.springframework.ai.chat.client.advisor=DEBUG
```

### Advisor Name and Order
- **Name**: Derived from class name if not explicitly set.
- **Order**: Controls execution sequence in the advisor chain.
    - Higher order number = executed **first**.
    - Default order = `0` (plain constructor).
    - Custom order: `new SimpleLoggerAdvisor(order)`.
    - Same-order advisors: execution sequence not guaranteed.

## 3. SafeguardAdvisor

### Purpose
- Blocks LLM requests if user input contains **sensitive words**.

### How It Works
1. Developer provides a list of sensitive words when creating the advisor.
2. Advisor checks incoming request for sensitive words.
3. If found: Request is **blocked**; default failure response is returned.
4. If not found: Request is forwarded to the LLM.

### Default Failure Response
> "I'm unable to respond to that due to sensitive content. Could we rephrase or discuss something else?"

### Use Case
- Common in enterprise environments for content safety and compliance.

## 4. Testing SimpleLoggerAdvisor

### Console Log Output (Request)
- System message (from `defaultSystem()` or `system()`).
- User message with role type = `user`.
- Default model name and parameters (temperature, context, streamUsage).

### Console Log Output (Response)
- Message type = `assistant`.
- Response ID (OpenAI-specific).
- Full response content.
- Metadata (tokens used, finish reason, etc.).

## 5. Per-Request Advisor Configuration

### Using `.advisors()` on Prompt Chain
```java name=ChatController.java
return chatClient.prompt()
    .system(systemMessage)
    .user(userMessage)
    .advisors(new SimpleLoggerAdvisor())     // single advisor
    // OR
    .advisors(List.of(new SimpleLoggerAdvisor(), new SafeguardAdvisor(sensitiveWords)))
    // OR
    .advisors(advisorSpec -> advisorSpec     // lambda for dynamic params
        .advisor(new SimpleLoggerAdvisor())
        .param("key", "value"))
    .call()
    .content();
```

### When to Use Per-Request vs. Default
| Configuration | When to Use |
|--------------|-------------|
| `defaultAdvisors()` on builder | Apply to **all** requests through this ChatClient |
| `.advisors()` on prompt | Apply to **specific** requests only |

## 8. Summary

| Concept | Key Point |
|---------|-----------|
| Default Advisor | Configured on ChatClient builder; applies to all requests |
| Per-Request Advisor | Configured on prompt chain; applies to specific request |
| SimpleLoggerAdvisor | Logs requests and responses at DEBUG level |
| SafeguardAdvisor | Blocks sensitive word inputs |
| Advisor Order | Higher number = executed first; same order = undefined sequence |
| Advisor Interfaces | CallAdvisor (sync), StreamAdvisor (streaming) |
| Debug Logging | Must enable `logging.level...=DEBUG` in properties |
