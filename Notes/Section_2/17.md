# Detailed Notes on Prompt Stuffing in Spring AI

## 1. The Problem: LLM Knowledge Limitations

### What LLMs Know
- Trained on massive amounts of **publicly available data**.
- Can answer general questions powerfully and accurately.

### What LLMs Don't Know
- **Private/proprietary data**: Company policy documents, internal databases, cloud storage.
- **Organization-specific information**: Custom HR policies, internal pricing, confidential data.
- LLMs were never trained on this private data → can only give **generic responses**, not accurate ones.

### Example
- Q: "How many paid leaves does my company provide?"
- Without context: LLM gives a generic answer (e.g., "typically 10-20 days").
- With company-specific data: LLM gives precise answer (e.g., "18 days as per your company policy").

---

## 2. What Is Prompt Stuffing?

### Definition
- **Prompt Stuffing** = Giving the LLM an "open book" before answering a question.
- Include **contextual data** or **reference text** along with the user's question.
- The LLM uses this extra content to answer accurately, even if not pre-trained on the topic.

### Also Known As
- **In-context learning**
- **Retrieval-Augmented Prompting** (when done programmatically)

### How It Works
1. Developer includes relevant context in the **System message**.
2. End user sends their question as the **User message**.
3. LLM combines both to generate an accurate, context-aware response.

### Example
```
System Message (Prompt Stuffing):
"According to the company's HR policy, employees are eligible for 18 days of paid 
leave annually. Unused leaves (up to 8) can be carried forward to the next year."

User Message:
"How many paid leaves do employees get each year?"

LLM Response:
"Employees get 18 days of paid leave annually. Up to 8 unused leaves can be 
carried forward to the next year."
```

---

## 3. When to Use Prompt Stuffing

| Scenario | Recommendation |
|----------|---------------|
| Small amount of context (few lines to ~200 lines) | ✅ Use Prompt Stuffing |
| Large documents (100s of pages) | ❌ Use RAG (Retrieval Augmented Generation) instead |

## 4. Demo Implementation

### Project Setup
- Create `PromptStuffingController` (copy of `PromptTemplateController`).
- Endpoint: `GET /api/prompt-stuffing`
- Parameter: `@RequestParam String message`

### System Prompt Template
- Create `systemPromptTemplate.st` under `resources/promptTemplates/`.
- Content (static; no dynamic placeholders):

```plaintext name=src/main/resources/promptTemplates/systemPromptTemplate.st
You are an internal HR assistant. You assist employees with queries related to 
HR policies only such as leave entitlements, working hours, benefits, and code of conduct.

Company HR Policy Details:
- Paid Leaves: 18 days annually
- Leave Carry Forward: Up to 8 unused leaves can be carried over to the next year
- Working Hours: [Standard hours defined here]
- Notice Period: [Details here]
- Paternity Leave: [Details here]
- Maternity Leave: [Details here]
[Other policy details...]

If an employee asks about anything outside HR policy, respond that you can only 
assist with HR-related queries.
```

### Controller Code
```java name=PromptStuffingController.java
@RestController
@RequestMapping("/api")
public class PromptStuffingController {

    private final ChatClient chatClient;

    @Value("classpath:promptTemplates/systemPromptTemplate.st")
    private Resource systemPromptTemplate;

    public PromptStuffingController(ChatClient.Builder chatClientBuilder) {
        this.chatClient = chatClientBuilder.build();
    }

    @GetMapping("/prompt-stuffing")
    public String promptStuffing(@RequestParam String message) {
        return chatClient.prompt()
                .system(systemPromptTemplate)  // static resource passed directly
                .user(message)
                .call()
                .content();
    }
}
```

- `system(Resource)`: Loads the entire `.st` file content as the system message.
- `user(String)`: Passes end user's question directly (no template needed here).

## 5. Testing

### Positive Scenario (With Prompt Stuffing)
- **Request**: `GET /api/prompt-stuffing?message=I used 5 leaves this year. How many will be forwarded to next year?`
- **Response**: "Since you used 5 leave days, you have 13 unused leave days remaining. Up to 8 unused leave days can be carried over to the next year. Therefore, you can carry over 8 leave days."
- LLM understands the twist in the question and answers accurately using provided context.

### Negative Scenario (Without Prompt Stuffing / Generic System Message)
- **Request**: Same as above.
- **Response**: "Please refer to the employee handbook or internal HR policy documents for accurate information."
- LLM lacks context → gives a generic, unhelpful response.

## 6. Dynamic vs. Static Prompt Stuffing

| Type | Description | Use Case |
|------|-------------|----------|
| **Static** | All context hardcoded in `.st` file | Simple scenarios; policy rarely changes |
| **Dynamic** | Load values (e.g., leave count, policies) from database at runtime via placeholders | Complex scenarios; frequently changing data |

- Example of dynamic stuffing: Load `{paidLeaves}`, `{noticeperiod}` from DB and inject into template at runtime using `.param()`.

---

## 7. Key Differences: With vs. Without Prompt Stuffing

| Without Prompt Stuffing | With Prompt Stuffing |
|------------------------|---------------------|
| LLM gives generic answers | LLM gives company-specific accurate answers |
| Redirects user to handbooks/docs | Directly answers based on provided context |
| Not useful for org-specific queries | Highly useful for internal tools |

---

## 8. Limitations of Prompt Stuffing

- **Only suitable for small amounts of data** (few lines to ~200 lines).
- **Not suitable for large documents** (100s of pages) due to:
    - Token limits (LLMs have maximum context window sizes).
    - Performance issues (large prompts = higher cost and latency).
    - Maintenance challenges.
- **Solution for large data**: Use **RAG (Retrieval Augmented Generation)** — covered in upcoming sections.

---

## 9. Summary

| Concept | Key Point |
|---------|-----------|
| Prompt Stuffing | Providing contextual data to LLM inside the system/user message |
| Purpose | Override LLM's lack of private/org-specific knowledge |
| Implementation | Load context in `.st` file; pass as `system(Resource)` |
| Best For | Limited context (small datasets, short policy documents) |
| Limitation | Not scalable for large documents; use RAG instead |
| Also Known As | In-context learning, Retrieval-Augmented Prompting |
