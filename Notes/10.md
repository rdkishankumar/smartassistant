# Notes on Integrating Spring AI with Cloud Providers (AWS Bedrock)

## Comparison of LLM Integration Options
- **OpenAI**: Zero setup, easy start; pay per usage (tokens). Locked to OpenAI models.
- **Local (Ollama/Docker)**: Setup required, infrastructure costs (servers); free AI usage, data private. Suitable for privacy-focused orgs.
- **Cloud Providers (AWS, Azure, GCP)**: Quick cloud setup, access to wide range of models (text, image, video generation). Integrates with existing infra. No vendor lock-in.

## Cloud Providers Overview
- AWS: Bedrock (models from Anthropic, Cohere, DeepSeek, Meta, Mistral, etc.; no OpenAI, as OpenAI is Microsoft-backed).
- Azure: Azure OpenAI Service.
- GCP: Google Vertex AI.
- Organizations choose based on priorities: ease vs. cost vs. privacy vs. model variety.

## Demo: Setting Up AWS Bedrock
- Prerequisites: AWS account with credit card (charges ~few cents for demo).
- Log in to AWS Console as root user.
- Select region (e.g., Mumbai for more models; Hyderabad limited).
- Search for "Bedrock" > Open service.
- Go to Model Catalog > Browse providers (Anthropic, Amazon, DeepSeek, etc.).
- Select model (e.g., Anthropic Claude Sonnet 4 – latest).
- Request access if disabled (Modify Access > Request Model Access). May be instant or 5-10 mins.
- Set up credentials via IAM:
    - Create Policy: Service = Bedrock, Actions = InvokeModel, Resources = All (wildcard; specify for prod).
    - Create User: Name = e.g., "spring-ai-demo".
    - Attach Policy to User.
    - Create Access Key: Choose "Local code" (not for prod).
    - Copy Access Key ID and Secret Access Key.
- Next: Configure in Spring AI application.
