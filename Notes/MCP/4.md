## Controller Implementation

### **MCP Client Controller**

Create `src/main/java/com/eazybytes/mcpclient/controller/MCPClientController.java`:

```java
package com.eazybytes.mcpclient.controller;

import org.springframework.ai.chat.client.ChatClient;
import org.springframework.ai.chat.client.advisor.SimpleLoggingAdvisor;
import org.springframework.web.bind.annotation.*;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

/**
 * REST Controller for MCP Client Chat API
 * 
 * Provides endpoints for interacting with LLM through MCP-discovered tools
 */
@RestController
@RequestMapping("/api/chat")
@CrossOrigin(origins = "*")
public class MCPClientController {

    private static final Logger logger = 
        LoggerFactory.getLogger(MCPClientController.class);

    private final ChatClient chatClient;

    /**
     * Constructor with ChatClient injection
     * 
     * @param chatClientBuilder Builder to create ChatClient
     */
    public MCPClientController(ChatClient.Builder chatClientBuilder) {
        logger.info("Initializing MCPClientController");
        
        // Build ChatClient with logging advisor
        this.chatClient = chatClientBuilder
            .defaultAdvisors(new SimpleLoggingAdvisor())
            .build();
        
        logger.info("ChatClient initialized with advisors");
    }

    /**
     * Simple chat endpoint without tools (baseline)
     * 
     * Tests basic LLM connectivity
     * 
     * @param message User message
     * @return LLM response
     */
    @PostMapping("/simple")
    public ChatResponse chat(@RequestBody ChatRequest request) {
        logger.info("Received chat request: {}", request.getMessage());
        
        try {
            String response = chatClient.prompt()
                .user(request.getMessage())
                .call()
                .content();
            
            logger.info("Chat response generated successfully");
            return new ChatResponse(response);
            
        } catch (Exception e) {
            logger.error("Error in chat endpoint", e);
            return new ChatResponse("Error: " + e.getMessage());
        }
    }

    /**
     * Chat endpoint with MCP-discovered tools
     * 
     * LLM can use tools discovered from MCP Server
     * 
     * @param message User message
     * @return Response (may include tool invocations)
     */
    @PostMapping("/with-tools")
    public ChatResponse chatWithTools(@RequestBody ChatRequest request) {
        logger.info("Received tool-enabled chat request: {}", 
                   request.getMessage());
        
        try {
            // ChatClient automatically includes MCP tools
            // through ToolCallbackProvider integration
            String response = chatClient.prompt()
                .user(request.getMessage())
                .call()
                .content();
            
            logger.info("Tool-enabled chat response generated");
            return new ChatResponse(response);
            
        } catch (Exception e) {
            logger.error("Error in tool-enabled chat", e);
            return new ChatResponse("Error: " + e.getMessage());
        }
    }

    /**
     * Health check endpoint
     */
    @GetMapping("/health")
    public HealthResponse health() {
        logger.debug("Health check requested");
        return new HealthResponse("UP");
    }

    /**
     * Request DTO
     */
    @lombok.Data
    @lombok.AllArgsConstructor
    @lombok.NoArgsConstructor
    public static class ChatRequest {
        private String message;
    }

    /**
     * Response DTO
     */
    @lombok.Data
    @lombok.AllArgsConstructor
    public static class ChatResponse {
        private String response;
    }

    /**
     * Health response DTO
     */
    @lombok.Data
    @lombok.AllArgsConstructor
    public static class HealthResponse {
        private String status;
    }
}
```

## ToolCallbackProvider Integration

### **Configuration with Tool Support**

Create `src/main/java/com/eazybytes/mcpclient/config/McpConfiguration.java`:

```java
package com.eazybytes.mcpclient.config;

import org.springframework.ai.chat.client.ChatClient;
import org.springframework.ai.chat.client.advisor.SimpleLoggingAdvisor;
import org.springframework.ai.mcp.client.api.ToolCallbackProvider;
import org.springframework.context.annotation.Bean;
import org.springframework.context.annotation.Configuration;
import org.slf4j.Logger;
import org.slf4j.LoggerFactory;

/**
 * Configuration for MCP Client integration
 * 
 * Configures ChatClient with MCP-discovered tools
 */
@Configuration
public class McpConfiguration {

    private static final Logger logger = 
        LoggerFactory.getLogger(McpConfiguration.class);

    /**
     * ChatClient Bean with MCP tool integration
     * 
     * ToolCallbackProvider automatically discovers tools from MCP Server
     * and injects them into ChatClient
     * 
     * @param chatClientBuilder ChatClient builder
     * @param toolCallbackProvider Tool callback provider (from MCP)
     * @return Configured ChatClient with tools
     */
    @Bean
    public ChatClient chatClient(
            ChatClient.Builder chatClientBuilder,
            ToolCallbackProvider toolCallbackProvider) {
        
        logger.info("Configuring ChatClient with MCP tools");
        
        ChatClient client = chatClientBuilder
            // Add logging advisor for debugging
            .defaultAdvisors(new SimpleLoggingAdvisor())
            // Add default tool callbacks from MCP Server
            .defaultToolCallbacks(toolCallbackProvider)
            .build();
        
        logger.info("ChatClient configured with MCP integration");
        return client;
    }

    /**
     * Alternative configuration with custom advisors
     * 
     * Uncomment to use custom configuration
     */
    /*
    @Bean
    public ChatClient chatClientCustom(
            ChatClient.Builder chatClientBuilder,
            ToolCallbackProvider toolCallbackProvider) {
        
        logger.info("Configuring ChatClient with custom advisors");
        
        return chatClientBuilder
            // Logging advisor
            .defaultAdvisors(new SimpleLoggingAdvisor())
            // Add more advisors as needed
            .defaultToolCallbacks(toolCallbackProvider)
            .build();
    }
    */
}
```

### **How ToolCallbackProvider Works**

```
MCP Server (Filesystem)
├─ Exposes tools
│  ├─ readFile()
│  ├─ writeFile()
│  ├─ listDirectory()
│  └─ ...
│
└─ STDIO Communication
       │
       ▼
MCP Client Component
├─ Discovers tools from server
├─ Creates ToolCallback objects
│
└─ ToolCallbackProvider Interface
   └─ Provides callbacks to ChatClient
       │
       ▼
ChatClient
├─ Receives tool callbacks
├─ Provides to LLM
│
└─ LLM can now use tools
   ├─ Recognizes need to use tool
   ├─ Calls tool via callback
   └─ Receives result
```

## Troubleshooting

### **Common Issues & Solutions**

#### **Issue 1: "MCP Server not found"**

```
Error: Cannot start MCP server: command not found

Solution:
1. Check Node.js installed: node --version
2. Check npx available: npx --version
3. Verify mcp-servers.json path
4. Check STDIO configuration

Command:
npm install -g npx  # If needed
```

#### **Issue 2: "Cannot read from directory"**

```
Error: Permission denied: /Users/eazybytes/mcp

Solution:
1. Create directory: mkdir -p /Users/eazybytes/mcp
2. Check permissions: ls -la /Users/eazybytes/mcp
3. Grant write permission: chmod 755 /Users/eazybytes/mcp
4. Update path in mcp-servers.json
```

#### **Issue 3: "OPENAI_API_KEY not found"**

```
Error: OpenAI API key not configured

Solution:
1. Set environment variable:
   export OPENAI_API_KEY="sk-..."
   
2. Or configure in IDE:
   Run → Edit Configurations → Environment variables
   
3. Or in application.properties:
   spring.ai.openai.api-key=sk-...
```

#### **Issue 4: "Port 8080 already in use"**

```
Error: Address already in use

Solution:
1. Change port in application.properties:
   server.port=8081
   
2. Or kill process on port 8080:
   lsof -i :8080  # macOS/Linux
   taskkill /PID <pid> /F  # Windows
```

#### **Issue 5: "MCP tools not discovered"**

```
Error: No tools available from MCP server

Solution:
1. Verify mcp-servers.json syntax (valid JSON)
2. Check directory exists and is accessible
3. Verify ToolCallbackProvider bean created
4. Check logs for MCP connection errors
5. Enable DEBUG logging for org.springframework.ai.mcp
```
