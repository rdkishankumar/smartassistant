## Overview

### **What is GitHub MCP Server?**

GitHub MCP Server is an **official MCP server** developed by GitHub that enables LLMs to:

```
GitHub Operations via Natural Language:
├─ Repository Management
│  ├─ Create repositories
│  ├─ List repositories
│  ├─ Get repository details
│  ├─ Update repository settings
│  └─ Delete repositories
│
├─ Code Management
│  ├─ Search code
│  ├─ Search repositories
│  ├─ Browse repository contents
│  └─ View file details
│
├─ Issue Management
│  ├─ Create issues
│  ├─ List issues
│  ├─ Update issues
│  ├─ Close issues
│  └─ Add comments
│
├─ Pull Requests
│  ├─ Create pull requests
│  ├─ List pull requests
│  ├─ Review pull requests
│  ├─ Merge pull requests
│  └─ View diffs
│
└─ Collaboration
   ├─ Assign issues
   ├─ Add reviewers
   ├─ Create milestones
   └─ Manage labels
```

### **Key Difference from Archived GitHub Server**

```
❌ AVOID: Archived GitHub MCP Server
└─ Listed under "Archived" in servers repo
   ├─ Outdated
   ├─ Not maintained
   ├─ May not work
   └─ Don't use for new projects

✅ USE: Official GitHub MCP Server
└─ Listed under main servers
   ├─ Officially maintained
   ├─ Latest features
   ├─ Full support
   └─ Recommended
```

## GitHub Personal Access Token

### **Creating Personal Access Token**

#### **Step 1: Access GitHub Settings**

```
1. Log in to GitHub: https://github.com
2. Click profile icon (top-right)
3. Click: Settings
4. Left sidebar: Developer settings
5. Click: Personal access tokens
6. Click: Fine grained tokens (or "Tokens (classic)")
```

#### **Step 2: Generate New Token**

```
Click: Generate new token
```

#### **Step 3: Configure Token**

```
Token Details:
├─ Token name: SpringAIDemo (or your choice)
├─ Description: Spring AI MCP Server Integration
├─ Resource owner: Your username
├─ Expiration: 30 days (or as preferred)
│  └─ ⚠️ Shorter expiration is more secure
│
└─ Permissions: Repository
   ├─ ☑ Read repository
   ├─ ☑ Read code
   ├─ ☑ Read issues
   ├─ ☑ Write issues
   ├─ ☑ Read pull requests
   ├─ ☑ Write pull requests
   ├─ ☑ Read repository contents
   ├─ ☑ Write repository contents
   └─ (Select based on your needs)
```

**⚠️ Security Note:** In production, grant only **minimum required permissions**

#### **Step 4: Create and Copy Token**

```
1. Click: Generate token
2. Token displayed (one-time only!)
3. Copy immediately: ghp_xxxxxxxxxxxxxxxxxxxxxxxx
4. Store securely: (NOT in code, use environment variables)
5. Can't retrieve later: If lost, generate new token
```

### **Token Security Best Practices**

```
❌ DON'T:
├─ Hardcode token in code
├─ Commit token to git
├─ Share token in chat/email
├─ Use same token everywhere
└─ Store in browser history

✅ DO:
├─ Store in environment variables
├─ Use .env files (but don't commit)
├─ Use secure vaults in production
├─ Rotate tokens regularly
├─ Use least-privilege permissions
└─ Delete unused tokens
```

## Docker Configuration

### **Why Docker for GitHub Server?**

```
Docker Benefits:
├─ No Node.js installation required
├─ Consistent environment
├─ Easy cleanup (removes container after use)
├─ Works across all OS
└─ Officially recommended method
```

### **Docker Prerequisites**

```
Required:
├─ Docker Desktop installed
│  ├─ macOS: https://www.docker.com/products/docker-desktop
│  ├─ Windows: https://www.docker.com/products/docker-desktop
│  └─ Linux: docker package
│
└─ Docker running
   ├─ Start Docker Desktop
   ├─ Verify: docker --version
   └─ Verify: docker run hello-world
```

### **Docker Command Breakdown**

```
docker run -i --rm -e GITHUB_PERSONAL_ACCESS_TOKEN=ghp_xxx \
  ghcr.io/modelcontextprotocol/github:latest

│     │   │  │                          │                     │
│     │   │  │                          │                     └─ Docker image
│     │   │  │                          └─ Environment variable
│     │   │  └─ Remove container after stop
│     │   └─ Interactive mode
│     └─ Run container
└─ Docker command
```

**Docker Arguments Explained:**

| Argument | Purpose | Example |
|----------|---------|---------|
| `run` | Create and run container | Required |
| `-i` | Interactive mode (stdin attached) | Required for tool testing |
| `--rm` | Remove container on exit | Clean up automatically |
| `-e VAR=value` | Environment variable | Pass token to server |
| `image:tag` | Docker image to run | `ghcr.io/modelcontextprotocol/github:latest` |

## mcp-servers.json Configuration

### **Multi-Server Configuration**

Create/Update `src/main/resources/mcp-servers.json`:

```json
{
  "mcpServers": {
    "filesystem": {
      "command": "npx",
      "args": [
        "-y",
        "@modelcontextprotocol/server-filesystem",
        "/Users/eazybytes/mcp"
      ],
      "env": {}
    },
    "github": {
      "command": "docker",
      "args": [
        "run",
        "-i",
        "--rm",
        "-e",
        "GITHUB_PERSONAL_ACCESS_TOKEN=${GITHUB_PERSONAL_ACCESS_TOKEN}",
        "ghcr.io/modelcontextprotocol/github:latest"
      ],
      "env": {}
    }
  }
}
```

### **Configuration Details**

```
filesystem server:
├─ Uses: npx (Node.js package executor)
├─ Package: @modelcontextprotocol/server-filesystem
├─ Monitor directory: /Users/eazybytes/mcp
└─ Env vars: None required

github server:
├─ Uses: docker (container engine)
├─ Image: ghcr.io/modelcontextprotocol/github:latest
├─ Args: run, -i, --rm, -e (environment)
└─ Env vars: GITHUB_PERSONAL_ACCESS_TOKEN (required)
```

### **Setting Environment Variable**

**macOS/Linux:**

```bash
# Set temporarily (current session)
export GITHUB_PERSONAL_ACCESS_TOKEN="ghp_your_token_here"

# Verify
echo $GITHUB_PERSONAL_ACCESS_TOKEN
```

**Windows PowerShell:**

```powershell
# Set temporarily
$env:GITHUB_PERSONAL_ACCESS_TOKEN="ghp_your_token_here"

# Verify
echo $env:GITHUB_PERSONAL_ACCESS_TOKEN
```

**Permanent Setup (macOS/Linux):**

```bash
# Add to ~/.bash_profile or ~/.zshrc
export GITHUB_PERSONAL_ACCESS_TOKEN="ghp_your_token_here"

# Then reload
source ~/.bash_profile
# or
source ~/.zshrc
```

**Application Properties:**

```properties
# src/main/resources/application.properties
spring.ai.mcp.github.token=${GITHUB_PERSONAL_ACCESS_TOKEN}
```

## Spring AI Integration

### **Application Properties Configuration**

```properties
# application.properties

# ========================================
# MCP Client Configuration
# ========================================
spring.ai.mcp.client.stdio.enabled=true
spring.ai.mcp.client.stdio.server-configurations=file:mcp-servers.json
spring.ai.mcp.client.enabled=true

# Logging
logging.level.org.springframework.ai.mcp=DEBUG
logging.level.org.springframework.ai=INFO
```

### **Automatic Tool Discovery**

```
Spring AI Startup Flow:

1. Application starts
   └─ Reads mcp-servers.json

2. For each server
   ├─ Filesystem:
   │  └─ Starts npx process
   │     └─ Lists tools: read_file, write_file, etc.
   │
   └─ GitHub:
      └─ Starts docker container
         └─ Lists tools: create_repository, list_issues, etc.

3. ToolCallbackProvider collects all tools
   └─ Injects into ChatClient

4. LLM receives all available tools
   └─ Can use any tool via natural language
```

### **No Code Changes Needed**

When you add servers to `mcp-servers.json`:

```java
// Same code as before - NO CHANGES!

@RestController
@RequestMapping("/api/chat")
public class ChatController {
    
    @PostMapping("/with-tools")
    public ChatResponse chatWithTools(@RequestBody ChatRequest request) {
        // Same implementation
        String response = chatClient.prompt()
            .user(request.getMessage())
            .call()
            .content();
        
        return new ChatResponse(response);
        // LLM now has tools from BOTH servers!
    }
}
```

## GitHub Tools Reference

### **Repository Management Tools**

```
create_repository
├─ Create new repository
├─ Parameters:
│  ├─ owner: Username (required)
│  ├─ name: Repo name (required)
│  ├─ description: Description (optional)
│  ├─ private: Boolean (optional)
│  └─ initialize_with_readme: Boolean (optional)
└─ Returns: Repository details

list_repositories
├─ List user's repositories
├─ Parameters:
│  ├─ owner: Username (required)
│  ├─ query: Search query (optional)
│  ├─ limit: Max results (optional)
│  └─ sort: Sort by (optional)
└─ Returns: List of repositories

get_repository
├─ Get specific repository details
├─ Parameters:
│  ├─ owner: Username (required)
│  └─ repo: Repository name (required)
└─ Returns: Repository details
```

### **Issue Management Tools**

```
create_issue
├─ Create new issue
├─ Parameters:
│  ├─ owner: Username (required)
│  ├─ repo: Repository name (required)
│  ├─ title: Issue title (required)
│  ├─ body: Issue description (optional)
│  ├─ assignee: Assignee username (optional)
│  └─ labels: List of labels (optional)
└─ Returns: Issue details

list_issues
├─ List repository issues
├─ Parameters:
│  ├─ owner: Username (required)
│  ├─ repo: Repository name (required)
│  ├─ state: open/closed/all (optional)
│  ├─ assignee: Filter by assignee (optional)
│  └─ limit: Max results (optional)
└─ Returns: List of issues

update_issue
├─ Update existing issue
├─ Parameters:
│  ├─ owner: Username (required)
│  ├─ repo: Repository name (required)
│  ├─ issue_number: Issue number (required)
│  ├─ state: open/closed (optional)
│  ├─ title: New title (optional)
│  └─ body: New body (optional)
└─ Returns: Updated issue
```

## Best Practices

### **Security Practices**

```
Token Security:
✅ DO:
├─ Use environment variables
├─ Rotate tokens periodically
├─ Use minimal permissions
├─ Store in secure vault (production)
└─ Audit token usage

❌ DON'T:
├─ Hardcode tokens
├─ Commit to git
├─ Share in logs
├─ Use same token everywhere
└─ Keep token indefinitely
```

### **API Usage Guidelines**

```
Prompt Guidelines:
✅ CLEAR AND SPECIFIC:
├─ "List my GitHub repositories"
├─ "Create repository named MyProject"
├─ "Create issue in org/repo with title..."
└─ "Create PR from feature branch to main"

❌ VAGUE:
├─ "Do GitHub stuff"
├─ "Make a repo"
├─ "Fix the issue"
└─ "Update something"

CONTEXT IMPORTANT:
├─ Always specify: owner/repo
├─ Include details: Create with description
├─ Mention targets: Branch names for PR
└─ Be explicit: repository names
```

### **Error Handling**

```
Common Errors and Solutions:

"Authentication failed"
└─ Solution: Check token validity and permissions

"Repository not found"
└─ Solution: Verify owner and repo names

"Permission denied"
└─ Solution: Check token has required permissions

"Rate limit exceeded"
└─ Solution: Wait or use GitHub API rate limit extensions

"Container not running"
└─ Solution: Verify Docker is running
   └─ Start Docker Desktop or `docker start container_name`
```

### **Performance Optimization**

```
Optimization Tips:
├─ Cache repository lists
├─ Batch operations when possible
├─ Use specific queries instead of listing all
├─ Limit results with pagination
└─ Combine tools efficiently in prompts

Example Optimization:
❌ "List all my repos, then search each one"
✅ "Search repositories for keyword"
```
