## MCP Overview

### **Definition**
MCP (Model Context Protocol) is an **open, standardized protocol** that:
- Centralizes all context (tools, prompts, resources) for AI applications
- Enables LLMs to discover and invoke capabilities in a standardized way
- Abstracts away implementation details of backend systems
- Works with any LLM provider (OpenAI, Anthropic Claude, Google Gemini, etc.)

### **Core Mission**
> **Standardize how AI applications access tools, prompts, and resources**

Think of MCP as a **bridge** that connects:
- **What LLMs need** (tools, prompts, context)
- **What applications have** (databases, APIs, files, services)

#### **Definition**
A **central registry and capability provider** that exposes tools, prompts, and resources in a standardized way.

#### **Responsibilities**
```java
MCP Server:
├─ Tool Registry
│  ├─ Tool definitions
│  ├─ Parameter schemas
│  ├─ Return types
│  └─ Documentation
│
├─ Tool Execution
│  ├─ Route requests to implementations
│  ├─ Validate parameters
│  ├─ Handle errors
│  └─ Return results
│
├─ Resource Management
│  ├─ Authentication/Authorization
│  ├─ Rate limiting
│  ├─ Logging/Auditing
│  └─ Version management
│
└─ Communication
   ├─ Respond to discovery requests
   ├─ Handle tool invocations
   └─ Manage connections
```

#### **Implementation Example**

```java
/**
 * MCP Server: Exposes task management capabilities
 */
@RestController
@RequestMapping("/mcp")
public class TaskManagementMcpServer {

    /**
     * DISCOVERY ENDPOINT
     * Called by MCP Client to get available tools
     */
    @GetMapping("/tools")
    public McpToolsResponse listTools() {
        return new McpToolsResponse(Arrays.asList(
            new McpToolDefinition(
                "createTask",
                "Create a new task with title and description",
                new McpToolSchema(
                    Map.of(
                        "title", Map.of("type", "string"),
                        "description", Map.of("type", "string")
                    ),
                    List.of("title")
                )
            ),
            new McpToolDefinition(
                "listTasks",
                "List all tasks",
                new McpToolSchema(new HashMap<>(), new ArrayList<>())
            ),
            new McpToolDefinition(
                "completeTask",
                "Mark a task as complete",
                new McpToolSchema(
                    Map.of("taskId", Map.of("type", "string")),
                    List.of("taskId")
                )
            )
        ));
    }

    /**
     * TOOL INVOCATION ENDPOINT
     * Called by MCP Client to execute a tool
     */
    @PostMapping("/tools/call")
    public McpToolResponse callTool(@RequestBody McpToolRequest request) {
        String toolName = request.getToolName();
        Map<String, Object> params = request.getParameters();

        switch (toolName) {
            case "createTask":
                return handleCreateTask((String) params.get("title"),
                                       (String) params.get("description"));
            case "listTasks":
                return handleListTasks();
            case "completeTask":
                return handleCompleteTask((String) params.get("taskId"));
            default:
                return new McpToolResponse("error", "Unknown tool: " + toolName);
        }
    }

    private McpToolResponse handleCreateTask(String title, String description) {
        // Business logic: Create task in database
        Task task = new Task(UUID.randomUUID().toString(), title, description);
        taskRepository.save(task);
        
        return new McpToolResponse("success",
            "Task created: " + task.getId());
    }

    private McpToolResponse handleListTasks() {
        List<Task> tasks = taskRepository.findAll();
        return new McpToolResponse("success",
            "Found " + tasks.size() + " tasks");
    }

    private McpToolResponse handleCompleteTask(String taskId) {
        Task task = taskRepository.findById(taskId).orElse(null);
        if (task != null) {
            task.setCompleted(true);
            taskRepository.save(task);
            return new McpToolResponse("success", "Task completed");
        }
        return new McpToolResponse("error", "Task not found");
    }
}
```

```java
    private final ChatClient chatClient;
    private final McpClientService mcpClient;
    private final PermissionService permissionService;
    private static final Logger logger = LoggerFactory.getLogger(ChatHostService.class);

    public ChatHostService(ChatClient chatClient, McpClientService mcpClient,
                          PermissionService permissionService) {
        this.chatClient = chatClient;
        this.mcpClient = mcpClient;
        this.permissionService = permissionService;
    }

    /**
     * Main chat method: User -> Host -> LLM -> Tool -> Response
     */
    public String processUserQuery(String userId, String userQuery) {
        logger.info("Host received query from user {}: {}", userId, userQuery);

        try {
            // STEP 1: Check permissions
            if (!permissionService.hasAccess(userId)) {
                logger.warn("User {} not authorized", userId);
                return "Sorry, you don't have access to this service.";
            }

            // STEP 2: Fetch available tools from MCP Server (via MCP Client)
            List<McpToolDefinition> availableTools = mcpClient.discoverTools();
            logger.info("Host discovered {} tools from MCP Server", availableTools.size());

            // STEP 3: Send to LLM with tools available
            String llmResponse = chatClient.prompt()
                .user(userQuery)
                .tools(convertToSpringAiTools(availableTools))
                .call()
                .content();

            // STEP 4: Parse LLM response for tool calls
            if (llmResponse.contains("tool_call")) {
                // LLM decided to use a tool
                String toolName = extractToolName(llmResponse);
                Map<String, Object> params = extractParameters(llmResponse);

                logger.info("LLM requested tool: {}", toolName);

                // STEP 5: Invoke tool via MCP Client
                String toolResult = mcpClient.invokeTool(toolName, params);
                logger.info("Tool result: {}", toolResult);

                // STEP 6: Send tool result back to LLM for final response
                String finalResponse = chatClient.prompt()
                    .user("Tool executed. Result: " + toolResult)
                    .call()
                    .content();

                return finalResponse;
            }

            return llmResponse;

        } catch (Exception e) {
            logger.error("Error processing query", e);
            return "Error: " + e.getMessage();
```

#### **Implementation Example**

```java
/**
 * MCP Client: Handles all communication with MCP Server
 * 
 * Sits inside the Host (Spring AI App)
 * Responsible for:
 * 1. Discovering tools from MCP Server
 * 2. Invoking tools on demand
 * 3. Handling errors and retries
 */
@Service
public class McpClientService {

    private final RestTemplate restTemplate;
    private final String mcpServerUrl = "https://mcp-server.example.com/mcp";
    private static final Logger logger = LoggerFactory.getLogger(McpClientService.class);

    // Cache for tool definitions
    private List<McpToolDefinition> toolCache;
    private long lastDiscoveryTime;
    private static final long CACHE_DURATION_MS = 300000; // 5 minutes

    public McpClientService(RestTemplate restTemplate) {
        this.restTemplate = restTemplate;
    }

    /**
     * Discover available tools from MCP Server
     * Uses caching to avoid frequent requests
     */
    public List<McpToolDefinition> discoverTools() {
        // Check if cache is still valid
        if (toolCache != null && 
            System.currentTimeMillis() - lastDiscoveryTime < CACHE_DURATION_MS) {
            logger.debug("Returning cached tools");
            return toolCache;
        }

        try {
            logger.info("Discovering tools from MCP Server: {}", mcpServerUrl);

            // Make discovery request to MCP Server
            McpDiscoveryResponse response = restTemplate.getForObject(
                mcpServerUrl + "/tools",
                McpDiscoveryResponse.class
            );

            if (response != null) {
                toolCache = response.getTools();
                lastDiscoveryTime = System.currentTimeMillis();

                logger.info("Successfully discovered {} tools", toolCache.size());
                toolCache.forEach(tool ->
                    logger.debug("Tool: {} - {}", tool.getName(), tool.getDescription())
                );

                return toolCache;
            }

        } catch (RestClientException e) {
            logger.error("Failed to discover tools from MCP Server", e);
            throw new McpClientException("Tool discovery failed", e);
        }

        return new ArrayList<>();
    }

    /**
     * Invoke a specific tool on the MCP Server
     */
    public String invokeTool(String toolName, Map<String, Object> parameters) {
        try {
            logger.info("Invoking tool: {} with parameters: {}", toolName, parameters);

            // Validate tool exists
            McpToolDefinition tool = findTool(toolName);
            if (tool == null) {
                throw new McpClientException("Tool not found: " + toolName);
            }

            // Validate parameters against schema
            validateParameters(tool, parameters);

            // Create tool invocation request
            McpToolInvocationRequest request = new McpToolInvocationRequest(
                toolName,
                parameters
            );

            // Send to MCP Server
            McpToolInvocationResponse response = restTemplate.postForObject(
                mcpServerUrl + "/tools/call",
                request,
                McpToolInvocationResponse.class
            );

            if (response != null) {
                logger.info("Tool execution successful");
                return response.getResult();
            }

        } catch (RestClientException e) {
            logger.error("Network error invoking tool", e);
            throw new McpClientException("Tool invocation failed", e);
        } catch (McpValidationException e) {
            logger.error("Validation error: {}", e.getMessage());
            throw e;
        }

        return "Tool execution failed";
    }

    /**
     * Find tool definition by name
     */
    private McpToolDefinition findTool(String toolName) {
        discoverTools(); // Ensure cache is populated
        return toolCache.stream()
            .filter(tool -> tool.getName().equals(toolName))
            .findFirst()
            .orElse(null);
    }

    /**
     * Validate parameters against tool schema
     */
    private void validateParameters(McpToolDefinition tool,
                                   Map<String, Object> parameters) {
        McpToolSchema schema = tool.getSchema();

        // Check required parameters
        for (String requiredParam : schema.getRequired()) {
            if (!parameters.containsKey(requiredParam)) {
                throw new McpValidationException(
                    "Missing required parameter: " + requiredParam
                );
            }
        }

        // Validate parameter types (simplified)
        for (String paramName : parameters.keySet()) {
            if (!schema.getProperties().containsKey(paramName)) {
                logger.warn("Unknown parameter: {}", paramName);
            }
        }
    }

    /**
     * Get tool definition with retry logic
     */
    public McpToolDefinition getToolWithRetry(String toolName, int maxRetries) {
        for (int i = 0; i < maxRetries; i++) {
            try {
                McpToolDefinition tool = findTool(toolName);
                if (tool != null) {
                    return tool;
                }

                // Refresh cache and retry
                toolCache = null;
                Thread.sleep(100 * (i + 1)); // Exponential backoff

            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
                break;
            }
        }

        throw new McpClientException("Tool not found after " + maxRetries + " retries");
    }
}

// Custom Exceptions
class McpClientException extends RuntimeException {
    public McpClientException(String message, Throwable cause) {
        super(message, cause);
    }
    
    public McpClientException(String message) {
        super(message);
    }
}

class McpValidationException extends RuntimeException {
    public McpValidationException(String message) {
        super(message);
    }
}
```

## Security & Access Control

### **Authentication & Authorization in MCP**

```java
/**
 * MCP Server Security Implementation
 */
@Configuration
public class McpSecurityConfiguration {

    /**
     * Secure MCP endpoints with authentication
     */
    @Bean
    public SecurityFilterChain mcpSecurityFilter(HttpSecurity http) 
            throws Exception {
        
        return http
            // MCP endpoints require API key
            .authorizeRequests()
                .antMatchers("/mcp/**")
                .hasRole("MCP_CLIENT")
            .and()
            // Use API key authentication
            .addFilter(new ApiKeyAuthenticationFilter())
            // Disable CSRF (for API)
            .csrf()
                .disable()
            .build();
    }

    /**
     * API Key Authentication Filter
     */
    public class ApiKeyAuthenticationFilter
            extends OncePerRequestFilter {

        @Override
        protected void doFilterInternal(
                HttpServletRequest request,
                HttpServletResponse response,
                FilterChain filterChain)
                throws ServletException, IOException {

            String apiKey = request.getHeader("X-API-Key");

            if (apiKey != null && isValidApiKey(apiKey)) {
                // Set authentication
                Authentication auth = new ApiKeyAuthentication(apiKey);
                SecurityContextHolder.getContext()
                    .setAuthentication(auth);
            }

            filterChain.doFilter(request, response);
        }

        private boolean isValidApiKey(String apiKey) {
            // Validate against database or environment variable
            String validKey = System.getenv("MCP_API_KEY");
            return validKey != null && validKey.equals(apiKey);
        }
    }
}

/**
 * Tool-level permission checks
 */
@Service
public class ToolPermissionService {

    public boolean canExecute(String userId, String toolName) {
        // Check if user has permission to execute tool
        User user = userRepository.findById(userId).orElse(null);
        if (user == null) return false;

        // Check role-based access
        if (toolName.equals("deleteRepository")) {
            return user.hasRole("ADMIN");
        }

        if (toolName.equals("createRepository")) {
            return user.hasRole("DEVELOPER");
        }

        // Default: allow
        return true;
    }

    public void logToolExecution(String userId, String toolName,
                                Map<String, Object> params,
                                boolean success) {
        // Audit logging for compliance
        auditLog.log(new ToolExecutionEvent(
            userId, toolName, params, success,
            LocalDateTime.now()
        ));
    }
}
```

## Interview Questions

### **Q1: Explain the three core components of MCP architecture**

**Answer:**

1. **MCP Host** - The orchestrator (Spring AI app, chatbot, IDE)
    - Receives user prompts
    - Manages permissions and sessions
    - Coordinates with LLM and MCP Client
    - Decides when/which tools to invoke

2. **MCP Client** - The connector (embedded in Host)
    - Discovers tools from MCP Server
    - Sends tool invocation requests
    - Handles errors and retries
    - Manages connection/caching

3. **MCP Server** - The capability provider
    - Registers and exposes tools
    - Executes tool logic
    - Manages authentication/authorization
    - Returns results

---

### **Q2: How does MCP differ from simply using REST APIs for tools?**

**Answer:**

| Aspect | REST API | MCP |
|--------|----------|-----|
| **Understanding** | Generic HTTP | AI-specific protocol |
| **Discovery** | Manual docs | Automatic registry |
| **Schema** | Varies | Standardized |
| **LLM Integration** | Manual | Native support |
| **Standardization** | None | Cross-vendor standard |

MCP is like REST for AI tools—it adds semantic understanding and standardization.

---

### **Q3: Why is MCP necessary when HTTP already exists?**

**Answer:**
HTTP is a **generic transport protocol** that doesn't understand AI concepts:
- Tools
- Prompts
- Resources
- Context

MCP is a **domain-specific protocol** built on top of HTTP that adds AI semantics, making integration:
- More efficient
- More standardized
- More discoverable
- More reliable

Like how GraphQL adds a query layer to HTTP, MCP adds an AI context layer.

---

### **Q4: Describe the flow when a user asks a question to an MCP-enabled chatbot**

**Answer:**

1. User enters: "Create a GitHub repo"
2. Host receives query, checks permissions
3. Host requests tools from MCP Client
4. MCP Client fetches from MCP Server
5. Host passes query + tools to LLM
6. LLM recognizes "need createRepository tool"
7. LLM returns tool invocation instruction
8. Host invokes via MCP Client
9. MCP Client sends to MCP Server
10. MCP Server executes (calls GitHub API)
11. MCP Server returns result
12. Host sends result back to LLM
13. LLM generates final response
14. User sees: "Repository created!"

### **Q5: How does MCP enable tool reusability?**

**Answer:**

**Before MCP:**
```
Tool locked in App 1
  ↓
Only App 1 can use it
  ↓
Tool can't be shared
```

**With MCP:**
```
Tool in MCP Server
  ↓
Any app/LLM can discover it
  ↓
Tool can be shared, monetized, versioned
  ↓
Ecosystem grows
```

MCP decouples tools from applications, enabling a true "tool economy."
