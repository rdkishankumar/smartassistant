# Notes on Integrating Spring AI with AWS Bedrock (Continued)

## Project Setup
- Create project "bedrock" (copy of Docker project).
- **POM.xml**: Change dependency to `spring-ai-starter-model-bedrock`.
- Sync Maven changes.

## Configuration
- **application.properties**:
    - Remove all OpenAI-related properties.
    - Add:
        - `spring.ai.bedrock.aws.region=ap-south-1` (e.g., Mumbai; get code from AWS console).
        - `spring.ai.bedrock.aws.access-key=<your_access_key>` (avoid hardcoding; use env vars in prod).
        - `spring.ai.bedrock.aws.secret-key=<your_secret_key>` (avoid hardcoding).
        - `spring.ai.model.chat=bedrock-converts` (default; optional for reference).
        - `spring.ai.bedrock.converse.chat.options.model=<inference_profile_id>` (from AWS Bedrock console).
    - Reference: Spring AI docs ([references/model/chat-models](https://docs.spring.io/spring-ai/reference/model/chat-models.html)) for properties.

- **Model ID Selection**:
    - In AWS Bedrock > Model Catalog > Select model (e.g., Anthropic Claude Sonnet 4).
    - Use **Inference Profile ID** from "Inference profiles" section (cross-region supported).
    - Avoid direct Model ID (may cause "invocation not supported" error; requires on-demand setup).

## Running the Application
- Rename main class to `BedrockApplication`.
- Build and run in debug mode.
- If startup fails due to region: Set env var `AWS_REGION=ap-south-1` in IntelliJ Run Configurations (potential bug workaround).
- Application starts successfully.

## Testing
- Use Postman (same as previous demos).
- Prompts:
    - "What is your name and which model you are using?" → "I'm Claude, an AI assistant created by Anthropic."
    - "Tell me a joke about Spring Boot" → Provides a joke.
- Confirms integration with AWS Bedrock LLM.
