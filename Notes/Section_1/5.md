# Notes on Building a Hello World Spring AI Application with OpenAI

## Project Setup
- Visit [start.spring.io](https://start.spring.io) to create a Spring Boot project.
- Select:
    - Language: Java
    - Build Tool: Maven
    - Spring Boot Version: Default
    - Group: com.easybytes
    - Artifact: openai
    - Description: (Any)
    - Package Name: com.easybytes.openapi
    - Packaging: JAR
    - Java Version: Default
- Add Dependencies:
    - Spring Web (for REST APIs)
    - Spring Boot DevTools (for fast restarts and live reload)
    - Spring AI OpenAI Starter (for OpenAI integration; other starters available for Anthropic, Mistral AI, Azure, Stability AI, etc.)
- Generate the project, download, and extract into a folder (e.g., section01 under spring-ai folder).
- Open in IDE (e.g., IntelliJ Ultimate or Community).

## Project Structure and Code
- **POM.xml**: Includes starters like spring-boot-starter-web, spring-boot-starter-devtools, spring-ai-starter-model-openai.
- Create package: `com.easybytes.openapi.controller`
- Create class: `ChatController.java`
    - Annotate with `@RestController` and `@RequestMapping("/api")`
    - Inject `ChatClient.Builder` via constructor (Spring auto-creates beans: `ChatModel` -> `OpenAiChatModel`, `ChatClient.Builder` -> `DefaultChatClientBuilder`)
    - Create `ChatClient` instance: `this.chatClient = chatClientBuilder.build();`
    - Method: `chat(String message)`
        - Annotate with `@GetMapping("/chat")`
        - Parameter: `@RequestParam("message") String message`
        - Return: `chatClient.prompt(message).call().content();`
        - This sends the message to OpenAI LLM and returns the response.

## Configuration
- **application.properties**:
    - Add logging pattern for better readability: `logging.pattern.console=%green(%time{HH:mm:ss}) %blue(%-5level) %red([%thread]) %yellow(%logger{36}) - %msg%n`
        - Displays time (green), log level (blue), thread (red), logger (yellow), and message.

## Running the Application
- Build and run: Application starts but throws exception: "OpenAI API key must be set"
- Reason: OpenAI models require an API key (not free).
- Bean creation fails for `OpenAiChatModel` due to missing API key.
- Next step: Set up OpenAI account and API key.

## Key Concepts
- Spring AI abstracts vendor complexities; switch providers easily.
- Fluent API for ChatClient: `prompt().call().content()`.
- Framework handles HTTP requests, endpoints, and security behind the scenes.
