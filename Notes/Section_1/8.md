# Notes on Integrating Ollama with Spring AI

## Introduction
- We've set up a Spring AI app with OpenAI LLMs, but some enterprises prefer local models for data privacy.
- Ollama: Open-source tool by Meta (backed by Facebook) to run LLMs locally on your infrastructure for free.
- No cloud charges; data remains private.

## Setting Up Ollama
- Download from [ollama.com](https://ollama.com) based on your OS.
- Install and start Ollama (run `ollama` in terminal; icon appears).
- Browse models (e.g., popular: Llama series by Meta).
- Choose a model: e.g., Llama 3.2:1b (1 billion parameters, 1.3GB, manageable locally).
    - Larger models (e.g., 404GB) need powerful servers (AWS, Azure, GCP).
- Download and set up: Run `ollama run <model>` (e.g., `ollama run llama3.2:1b`).
    - Downloads and installs locally.
- Test: Ask questions in terminal (e.g., "What is the capital of India?" → "New Delhi").

## Integrating Ollama into Spring AI
- Create a new project (copy of OpenAI project) named "Ollama".
- **POM.xml**: Replace `spring-ai-starter-model-openai` with `spring-ai-starter-model-ollama`.
- **application.properties**:
    - Remove OpenAI properties (e.g., API key).
    - Add:
        - `spring.ai.model.chat=ollama`
        - `spring.ai.ollama.chat.options.model=llama3.2:1b`
    - Optional: `spring.ai.ollama.base-url=http://localhost:11434` (default; change if Ollama is on another server).
- **Controller**: No changes needed (same `ChatController` code; not coupled to OpenAI).
    - Rename main class if desired (e.g., to `OllamaApplication`).
- Build and run: App connects to local Ollama LLM.

## Testing
- Use Swagger (same as OpenAI).
- Example prompts:
    - "What is your name and which model you are using?" → "I'm an artificial intelligence model known as Llama."
    - "Tell me a joke about Spring Boot" → Varies (model-dependent).
- Smaller models (e.g., 1B parameters) for quick demos/unit testing; larger for better responses.

## Benefits and Notes
- Run models on local machines/servers; no external sharing.
- Open-source; free.
- Use smaller models locally; scale up for production.
- Documented steps: Install Ollama → Run model command → Update dependencies/properties → Test API.
