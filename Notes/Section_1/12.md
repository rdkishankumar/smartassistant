# Notes on Multi-Model Support in Spring AI

## Introduction
- Real enterprise apps often use multiple ChatModels due to varying capabilities (e.g., text, image, video generation).
- No single model excels at all tasks; choose based on use cases.
- Demo: Spring AI app with Ollama and OpenAI models.

## Project Setup
- Create project "multi-model".
- **POM.xml**: Add starters for multiple models (e.g., `spring-ai-starter-model-openai`, `spring-ai-starter-model-ollama`).
- **application.properties**:
    - OpenAI: `spring.ai.openai.api-key=${OPENAI_API_KEY}`
    - Ollama: `spring.ai.ollama.chat.options.model=llama3.2:1b`
    - Disable auto ChatClient.Builder: `spring.ai.chat.client.enabled=false` (allows manual bean creation for multi-model).

## Configuration
- Create package `config`.
- Class: `ChatClientConfig` (@Configuration).
- Manually create ChatClient beans (one per model).
- Methods:
    - `openAiChatClient(ChatModel openAiChatModel)`: Return `ChatClient.create(openAiChatModel)`.
    - `ollamaChatClient(ChatModel ollamaChatModel)`: Return `ChatClient.builder(ollamaChatModel).build()`.
- Builder approach allows more control (e.g., `defaultSystem()`; discussed later).

## Controller
- Class: `MultiModelChatController` (@RestController, @RequestMapping("/api")).
- Endpoints:
    - `@GetMapping("/openai/chat")`: `openAiChat(ChatClient openAiChatClient, @RequestParam String message)`
    - `@GetMapping("/ollama/chat")`: `ollamaChat(ChatClient ollamaChatClient, @RequestParam String message)`
- Inject ChatClients with @Qualifier ("openAiChatClient", "ollamaChatClient").
- Logic: Return `chatClient.prompt(message).call().content()`.

## Testing
- Ensure Ollama model is running locally and OpenAI API key is set.
- Postman:
    - OpenAI endpoint: "What is your name..." → "I'm ChatGPT..."
    - Ollama endpoint: "What is your name..." → "I'm an artificial intelligence model known as Llama..."
- Confirms multi-model setup.
