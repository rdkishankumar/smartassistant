# Notes on Setting Up Local LLMs with Docker Model Runner

## Introduction
- Docker can containerize and run local LLM models, similar to other software.
- Alternative to Ollama for enterprises preferring local setups.
- Feature is in beta; may become stable.

## Requirements and Setup
- Install Docker Desktop (free for local development) from [docker.com](https://docker.com).
- Version Requirements:
    - Windows: Docker Desktop 4.41+, Nvidia GPU.
    - Mac: Docker Desktop 4.40+, Apple Silicon.
    - Linux: Specific kernel/hardware support.
- Enable Beta Feature:
    - Open Docker Desktop > Settings > Beta Features > Enable "Docker Model Runner".
    - Enable "Host side TCP support" on port 12434 (default for LLM models).
    - Apply and restart Docker Desktop.
- Check Installation: Run `docker model version` in terminal.

## Running a Model
- In Docker Desktop: Dashboard > Models > Local (see running models) > Docker Hub (browse available models).
- Choose a model: e.g., `ai/gemma3` (Google's Gemma3, 3.88B parameters, 2.3GB, lightweight for testing).
- Run: `docker model run ai/gemma3` (pulls and starts model if not local).
- Wait for download/setup; interactive chat starts in terminal.
- Test: Ask questions (e.g., "Who are you and how can you help me?").

## Integrating with Spring AI
- Create project "DockerModel" (copy of previous Ollama project).
- **POM.xml**: Use `spring-ai-starter-model-openai` (reuses OpenAI library for compatibility).
- **application.properties**:
    - `spring.ai.openai.chat.options.model=ai/gemma3`
    - `spring.ai.openai.api-key=<dummy_value>` (mandatory for bean creation, even for local models).
    - `spring.ai.openai.chat.base-url=http://localhost:12434/engines` (points to local Docker model; overrides OpenAI default).
- **Controller**: No changes (same `ChatController`).
- Build and run in debug mode.

## Testing
- Use Postman:
    - Prompt: "What is your name and which model you are using?"
        - Response: e.g., "I'm a large language model created by Google. I'm an open-weight model..."
    - Prompt: "Tell me a joke about Spring Boot" → Provides a joke.
- Confirms integration works.

## Stopping and Notes
- Stop model: Type `/bye` in terminal chat.
- Benefits: Local setup, no external data sharing.
- Documented in slides for reference.
- Use for future projects needing local LLMs.
